# Codex 交接说明

## 当前结论

当前最终默认策略是：

1. 原始 `竞价爬升` 信号生成候选池
2. 过滤 `竞价匹配金额_openapi >= 5000万`
3. 过滤 `实体涨跌幅昨日 < 实体涨跌幅前日`
4. 过滤 `市场20日高低差 >= 0`
5. 按 `个股热度排名昨日` 从小到大选前 3
6. `T` 日开盘等权买入
7. 从 `T+1` 开始，第一次收盘不封涨停就卖出

## 重要修复点

### 1. 市场宽度过滤曾经失效

主脚本之前有一个 bug：

- `新因子策略版` 虽然配置了 `market_filter={"min_high20_minus_low20": 0}`
- 但实际调用 `run_config_backtest()` 时传的是空的 `breadth_df`
- 所以市场过滤根本没有生效

这个 bug 已修复：

- 现在 `新因子策略版` 会传真实 `breadth_df`
- 修复后结果从 `3.5652` 提升到 `4.2908`

### 2. 补数过程中出现过回写覆盖问题

- 中途有一次把旧因子覆盖掉的 bug
- 已通过 `重建扩展因子文件.py` 从缓存重建修复

如果后续再补数，优先保留这个恢复脚本。

## 补数链路说明

为了补完整验证期因子，实际用了三条链路：

### 1. OpenAPI

- `wencai_openapi.py`
- `竞价因子补充.py`

特点：

- 前半段可用
- 后面容易 `403`
- 不是最后主力链路

### 2. unified-wap

- `wencai_unifiedwap.py`
- `竞价因子补充_unifiedwap.py`

特点：

- 结合浏览器态 `cookie + hexin-v` 低频补数
- 是后半段补数主力之一
- `hexin-v` 和 `Cookie` 有时效性
- 如果继续补新日期，需要重新刷新

### 3. pywencai 兜底

- `竞价因子补充_pywencai.py`

特点：

- 后半段最后一批日期主要靠它补完
- 同样依赖时效 cookie
- 适合做兜底，不适合作为唯一主链
