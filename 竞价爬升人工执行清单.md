# 竞价爬升人工执行清单

## 当前执行版策略

- 原始信号: `竞价爬升`
- 过滤 1: `竞价匹配金额_openapi >= 5000万`
- 过滤 2: `实体涨跌幅昨日 < 实体涨跌幅前日`
- 过滤 3: `实体涨跌幅前日 >= 0`
- 市场开关: `市场20日高低差 >= 0`
- 行业过滤: `T日申万一级行业开盘涨幅 > 0`
- 排序: 先按 `竞价未匹配占比` 降序, 再按 `竞昨成交比估算` 降序, 最后按 `个股热度排名昨日` 升序
- 买入: 开盘买前 `3` 只, 等权
- 卖出: 从 `T+1` 开始, 第一次收盘不封涨停就卖

## 盘前

### 1. 确认市场开关

- 查看 [竞价爬升-20240504-市场宽度.csv](C:/Users/zr/Documents/caution/竞价爬升-20240504-市场宽度.csv)
- 找到当天对应的 `市场20日高低差`
- 如果 `< 0`, 当天不做
- 如果 `>= 0`, 继续执行

### 2. 准备候选池

- 跑出原始 `竞价爬升` 候选股
- 保留每只股票的以下字段:
  - `股票代码`
  - `股票简称`
  - `竞价匹配金额_openapi`
  - `竞价涨幅今日`
  - `开盘价:不复权今日`
  - `实体涨跌幅昨日`
  - `实体涨跌幅前日`
  - `成交量昨日`
  - `个股热度排名昨日`

## 竞价阶段

### 3. 做硬过滤

- 只保留 `竞价匹配金额_openapi >= 5000万`
- 只保留 `实体涨跌幅昨日 < 实体涨跌幅前日`
- 只保留 `实体涨跌幅前日 >= 0`
- 只保留 `T日申万一级行业开盘涨幅 > 0`

### 4. 计算排序因子

- `昨收估算 = 开盘价:不复权今日 / (1 + 竞价涨幅今日 / 100)`
- `昨日成交额估算 = 昨收估算 × 成交量昨日`
- `竞昨成交比估算 = 竞价匹配金额_openapi / 昨日成交额估算`
- `竞价未匹配占比 = 竞价未匹配金额 / 竞价匹配金额_openapi`

理解方式:

- 这个值越大, 代表集合竞价阶段成交金额, 占昨天全天成交额的比例越高
- 它更适合拿来排序, 不建议直接当硬门槛

### 5. 排序并选股

- 第 1 排序键: `竞价未匹配占比` 降序
- 第 2 排序键: `竞昨成交比估算` 降序
- 第 3 排序键: `个股热度排名昨日` 升序
- 取前 `3` 只

## 开盘执行

### 6. 买入

- 时间: `9:30` 开盘后尽快执行
- 方式: 前 `3` 只等权买入
- 如果不足 `3` 只, 就买实际满足条件的数量
- 如果当天市场开关不满足, 空仓

## 持仓与尾盘

### 7. 持仓规则

- 买入当天不卖
- 从 `T+1` 开始, 每天检查是否继续封涨停

### 8. 卖出规则

- 如果当天收盘仍封涨停, 继续持有
- 如果当天收盘不封涨停, 当天卖出

实盘执行建议:

- 回测按收盘口径
- 实盘可在 `14:55` 左右确认是否仍封板
- 如果明显开板且尾盘无法回封, 直接卖出

## 每日最简流程

### 盘前 3 句话版

1. 看 `市场20日高低差`, 小于 0 不做
2. 候选池先过 `5000万竞价金额`、`实体昨日<前日`、`前日实体>=0`
3. 剩下按 `竞价未匹配占比 -> 竞昨成交比估算 -> 热度排名` 排序, 取前 3

### 收盘前 2 句话版

1. 只检查持仓是否还封涨停
2. 不封板就卖, 封板继续拿

## 市场20日高低差怎么获得

### 最省事的方式

  - `市场20日新低数`

### 它的定义

- `市场20日高低差 = 市场20日新高数 - 市场20日新低数`
