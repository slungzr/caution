## 核心逻辑

`竞价市值影响探索.py` 会：

- 默认读取因子文件：`竞价爬升-20240504-扩展因子-至20260423.csv`
- 加载原回测模块：`竞价爬升策略回测.py`
- 默认使用原策略配置：`AUCTION_RATIO_RANK_CONFIG`，也就是 `竞昨比排序版`
- 沿用原策略市场宽度过滤：
  `market_filter={"min_high20_minus_low20": 0}`
- 优先使用已有字段：`a股市值(不含限售股)昨日`
- 标准化为：`昨日a股流通市值`
- 如果市值缺失，用 Playwright 问财补齐并缓存

## 问财补齐口径

必须带年份，避免跨年歧义。

先按当日前一交易日热度前 100 补：

```text
2024年5月8日a股流通市值，2024年5月8日个股热度排名前100
```

如果热度前 100 没覆盖候选代码，再按候选代码精确补：

```text
600678、300719、600796，2024年5月8日a股流通市值
```

这个备用逻辑已经实现并验证过。

## 缓存机制

每个交易日缓存一个文件：

```text
cache/wencai_market_cap\YYYYMMDD.csv
```

其中 `YYYYMMDD` 是“策略买入日”，但问财查询的是它的“前一交易日”的流通市值。

例如：

- 缓存文件：`cache/wencai_market_cap\20240509.csv`
- 查询日期：`2024年5月8日a股流通市值`

## 补齐完成后跑回测

```powershell
python .\竞价市值影响探索.py --scan-mode both
```

`--scan-mode both` 会同时跑：

- 固定分档：
  - `<20亿`
  - `20-50亿`
  - `50-100亿`
  - `100-200亿`
  - `>=200亿`
- 阈值扫描：
  - `>=20亿`
  - `>=30亿`
  - `>=50亿`
  - `>=80亿`
  - `>=100亿`
  - `>=150亿`
  - `>=200亿`
  - `>=300亿`

## 重要注意事项

- 不要把当前短样本结论直接写进实盘策略。
- 补齐全历史市值后再看：
  - 是否收益随市值阈值提高而稳定改善
  - 是否只是少数交易日或少数票造成的偶然
  - 各档交易笔数是否足够
  - 市场宽度过滤是否仍然启用
- 当前脚本默认仍然考虑市场宽度：
  `市场20日新高数 - 市场20日新低数 >= 0`

## 可能遇到的问题

如果 Playwright 浏览器没装：

```powershell
python -m playwright install
```

或：

```powershell
python -m playwright install chrome
```

如果无头模式不稳定，先不要加 `--headless`。

如果问财返回验证码或登录，先手动打开同一个 profile 登录：

```text
.playwright_wencai_profile
```

如果 H 盘不可见，先确认网络盘或挂载盘恢复后再跑。
